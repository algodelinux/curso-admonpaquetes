Gestión de paquetes mediante apt

apt es una herramienta de gestión de paquetes que incluye, en un sólo ejecutable, los comandos más utilizados habitualmente de apt-cache y apt-get, mostrando una barra de progreso y añadiendo un poco de color a la salida de dichos comandos.
Podemos usar apt desde la línea de comandos. Ahora bien, si tratáis de usarlo en un script, veréis que os muestra un warning como el siguiente:
WARNING: apt does not have a stable CLI interface. Use with caution in scripts.
Es decir, que si queréis realizar operaciones de gestión de paquetes en un script, mejor usad apt-get o aptitude.

En este apartado, vamos a ver algunos ejemplos de uso de apt.

Por ejemplo, si queremos actualizar los índices de los repositorios, no tendremos más que hacer un:
   # apt update
Si queremos listar los paquetes instalados en el equipo, haremos un:
   # apt list --installed
Ahora bien, si queremos listar los paquetes para los cuales hay actualizaciones en los repositorios, haremos un:
   # apt list --upgradable
Si lo que queremos es realizar una actualización de paquetes, ejecutaremos:
   # apt upgrade
Si una actualización requiere desinstalar paquetes, al tratar de realizar un apt upgrade, el sistema nos informará de que hay paquetes retenidos. Si queremos realizar dicha actualización, tendremos que utilizar apt full-upgrade.
   # apt full-upgrade

Al ejecutar el comando anterior, el sistema nos informará de qué paquete o paquetes se va o van a desinstalar. Esa información nos servirá para decidir si nos interesa actualizarlo o no.

Si queremos buscar paquetes utilizaremos apt search con una expresión regular. Veamos un ejemplo:
   # apt search ^linex-*
El comando anterior buscará los paquetes que comienzan con la palabra linex, tienen a continuación un guión y cualquier número de caracteres. La lista de paquetes se mostrará ordenada por orden alfabético.

Si queremos obtener información acerca de un paquete o un conjunto de paquetes:
   # apt show ^linex-*
Con el comando anterior obtendríamos información de todos y cada uno de los paquetes que comienzan con la palabra linex, tienen a continuación un guión y cualquier número de caracteres.

Con apt install instalamos paquetes:
   # apt install linex-xubuntu-core
El paquete linex-xubuntu-core, en particular, instala todos los paquetes necesarios para configurar un entorno "linex" en los centros educativos.
Con apt remove desinstalamos paquetes. Por ejemplo, si queremos desinstalar el paquete linex-xubuntu-core, haremos lo siguiente:
   # apt remove linex-xubuntu-core
Aunque aún no hemos hablado de ello, linex-xubuntu-core es un metapaquete que permite instalar o desinstalar de un plumazo un conjunto de paquetes. De este modo, al desinstalar linex-xubuntu-core, se desinstalarán los paquetes relacionados en el paquete. No os preocupéis por ello porque ya lo veremos más adelante.

Si queremos buscar un paquete en los repositorios, haremos uso del subcomando search:
   # apt search fbreader
Si queremos ver en qué repositorios se encuentra un paquete y sus prioridades usaremos el subcomando policy:
   # apt policy fbreader

Si queremos descargar un paquete desde los repositorios, podemos ejecutar la siguiente combinación de comandos:

   # apt update && apt download fbreader
Como podéis observar, primero hacemos una actualización de índices de repositorios y, si todo ha ido bien, se realizará la descarga del paquete fbreader.

Seguramente, en alguna ocasión, habréis descargado paquetes desde algún lugar y luego los habéis instalado con dpkg. Ésto tiene un inconveniente: Que dpkg no resuelve dependencias, y, si el paquete que habéis instalado tiene dependencias, tras instalarlo con dpkg, habréis tenido que realizar un apt -f install para que se resuelvan las dependencias y se instalen los paquetes faltantes o se desinstale el paquete porque no existen las dependencias en los repositorios.

Por si no lo sabéis, apt permite instalar paquetes descargados previamente  y tiene una ventaja frente a dpkg: Que resuelve dependencias.

Para instalar un paquete mediante apt, que habéis descargado previamente, tendréis que especificar la ruta local donde se encuentra el paquete. Por ejemplo: Supongamos que habéis descargado el paquete AutoFirma_1_8_2.deb desde la página https://firmaelectronica.gob.es
   # apt -y install /root/AutoFirma_1_8_2.deb
Si no especificáis la ruta, el sistema de gestión de paquetes entenderá que queréis instalar el paquete desde los repositorios y el proceso de instalación fallará porque dicho paquete no se encuentra en los repositorios.
