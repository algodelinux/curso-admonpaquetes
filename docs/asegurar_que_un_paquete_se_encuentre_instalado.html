Asegurar que un paquete se encuentre instalado

Para  garantizar que un paquete se encuentre instalado mediante Puppet, tan sólo tenemos que utilizar un recurso package dentro de una clase.

Por ejemplo, supongamos que tenemos una clase llamada software y queremos garantizar que el paquete vlc se encuentre instalado en la máquina. Dentro de la clase software podríamos utilizar un recurso package de la siguiente manera:

   class software {

      package { 'vlc':
         ensure => 'present',
      }

   }

Al definir el recurso package anterior, le estamos indicando a Puppet que deseamos que garantice que el paquete vlc se encuentre presente. Y Puppet se encargará de utilizar el mecanismo que tenga por defecto el sistema operativo para garantizar que se encuentre instalado.

Ésta sería otra forma de indicar lo mismo:

   class software {

      package { 'vlc':
         ensure => 'installed',
      }

   }

Al indicar a Puppet que deseamos que un paquete se encuentre instalado, si comprueba que ya lo está, no hará nada más, aunque existan actualizaciones.

Si queremos decirle a puppet que garantice que un paquete se encuentre actualizado, tan sólo tenemos que usar el valor latest en el atributo ensure:

   class software {

      package { 'vlc':
         ensure => 'latest',
      }

   }

De este modo, si en algún momento existe una versión más reciente del paquete vlc, Puppet lo actualizará.

Por otro lado, como ya comentamos anteriormente, podemos indicarle a Puppet que garantice que se encuentra instalada una determinada versión de un paquete, simplemente especificando la versión entre llaves:

   class software {

      package { 'vlc':
         ensure => '{3.0.8-0ubuntu18.04.1}',
      }

   }

Como ya hemos dicho anteriormente, puppet se encargará de utilizar el mecanismo que tenga por defecto el sistema operativo para garantizar que se encuentre instalado. En el caso de Debian, el proveedor por defecto es apt. En el caso de Redhat es rpm. Si consultáis la referencia del recurso package, veréis que existen multitud de proveedores como dpkg, pip, gem, windows, etc... por mencionar algunos.

En un momento determinado, puede que nos interese cambiar el proveedor.

Por ejemplo:

   class software {

      package { 'vlc':
         ensure => 'latest',
         provider => 'dpkg',
         source => '/tmp/vlc.deb',
      }

}

En este caso, le estamos diciendo a Puppet que debe garantizar que se encuentre instalada la última versión del paquete vlc, que el mecanismo de instalación (proveedor) del paquete debe ser dpkg y que debe obtener el paquete desde /tmp/vlc.deb en el equipo cliente. 

Naturalmente, al utilizar este mecanismo, para que Puppet pueda garantizar que el paquete vlc se encuentre instalado, el paquete vlc.deb debe encontrarse en /tmp.

Por otro lado, podemos usar parámetros de instalación mediante el atributo install_options:

   class software {

      package { 'mysql-server-5.5':
         ensure => 'present',
         install_options => ['--allow-unauthenticated', '-f'],
      }

   }
Los parámetros de instalación que podemos especificar son los que el instalador tenga definidos.

Como ya hemos dicho anteriormente, el recurso package nos va a permitir garantizar que se encuentren instalado, o no, un determinado software en diferentes sistemas operativos porque, si no especificamos otra cosa, Puppet va a usar el mecanismo que tenga por defecto el sistema operativo para lograrlo.

Por ejemplo, podemos instalar un paquete software en Windows de la siguiente forma:
   package { 'Atom':
      ensure => installed,
      source => 'C:/Instaladores/AtomSetup-x64.exe',
   }

Con el recurso anterior, le estamos diciendo a Puppet que garantice que el paquete Atom, cuyo instalador es AtomSetup-x64.exe y se encuentra en C:/Instaladores/, se encuentre instalado. De este modo, si el paquete no se encuentra instalado en la máquina, Puppet lo instalará usando el instalador que le hemos indicado.

El proveedor por defecto en Windows es "windows".

Supongamos que tenemos configurado nuestro sistema para que se pueda utilizar Chocolatey como fuente de paquetes para Windows. En este caso, podríamos usar el proveedor de paquetes chocolatey:

   package { 'Atom':
      ensure => 'installed',
      provider => chocolatey,
   }

Con ésto, le estaríamos diciendo a Puppet que garantice que el paquete Atom se encuentre instalado y que para hacerlo, use el proveedor chocolatey. Lo que hará Puppet será buscar el paquete el el repositorio de Chocolatey para instalarlo.

Para poder usar Chocolatey como proveedor de paquetes, es necesario:

  • Que el servidor tenga instalado el módulo puppetlabs-chocolatey: puppet module install puppetlabs-chocolatey
  • Que en el servidor hagamos el include chocolatey en un fichero manifest.
  • Que los clientes Windows tengan instalado el paquete chocolatey.
