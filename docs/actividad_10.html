Iniciar un cliente puppet

Clona la máquina virtual "xubuntu" en una nueva máquina virtual.

Asigna el nombre "cliente" a la nueva máquina virtual y selecciona la opción "Generar nuevas direcciones MAC para todos los adaptadores de red". Realiza una clonación completa.

Una vez clonada, inicia la máquina virtual "cliente".

Abre un terminal en la máquina "cliente" y edita el fichero /etc/hosts ejecutando el comando: "sudo nano /etc/hosts"

Instalar el paquete audacity en clientes mediante puppet

Vamos a crear un recurso en el servidor puppet que instale el paquete audacity. Por tanto, sitúate en la máquina virtual puppetserver.

No vamos a entrar en detalles sobre la creación de módulos puppet. Como lo que nos interesa es tan sólo probar la gestión de paquetes en clientes mediante un servidor puppet, insertaremos los recursos que instalan paquetes directamente dentro del fichero /etc/puppetlabs/code/environments/production/manifests/site.pp

Edita el fichero /etc/puppetlabs/code/environments/production/manifests/site.pp

Verás que este fichero tan sólo contiene la definición del nodo default:

   node default {

   }

La palabra reservada "default" representa cualquier nodo. Es decir, todos los recursos que incluyamos en el nodo default se aplicarán a todos los clientes.
