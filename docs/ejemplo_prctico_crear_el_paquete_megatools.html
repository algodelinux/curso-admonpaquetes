Ejemplo práctico: Crear el paquete megatools

megatools es un conjunto de herramientas que permiten acceder al servicio mega.nz desde la línea de comandos.

Este conjunto de herramientas se suministran mediante código fuente que podéis descargar desde la siguiente URL. 

Instalar una herramienta desde código fuente no suele ser un problema si tan sólo vas a instalarlo en un equipo. Pero, si vas a instalarlo en un conjunto de máquinas de tu red, lo mejor es crear un paquete debian, subirlo a tu repositorio local e instalarlo desde allí.

En este apartado, vamos a ver cómo instalar megatools desde el código fuente y, al mismo tiempo, crear el paquete mediante checkinstall.

Para empezar, descargamos el código fuente en nuestro directorio home:

   $ wget https://megatools.megous.com/builds/megatools-1.10.3.tar.gz

Una vez descargado, descomprimimos el fichero:

   $ tar xfvz megatools-1.10.3.tar.gz

Se creará una un directorio  megatools-1.10.3.

Entramos dentro del directorio:

   $ cd megatools-1.10.3/

Una vez dentro, echamos un vistazo al fichero que contiene las instrucciones de instalación: INSTALL.

Por lo que he podido observar, las instrucciones de instalación son las habituales en el caso de software distribuidos mediante código fuente:

  1. Ejecutar el script ./configure para comprobar si la máquina dispone de todos los recursos para instalarlo.

    1. Si al ejecutarlo se produce algún error que nos dice que falta algún paquete, buscamos de qué paquete se trata, lo instalamos y volvemos a ejecutar ./configure de nuevo hasta que tengamos instalado todo el software que se necesita para compilar el código.

  2. Una vez que tengamos instalados todos los paquetes que se necesitan, ejecutamos make para compilar el código fuente.

  3. Una vez compilado, en lugar de ejecutar "make install" para instalar el software, ejecutamos "checkinstall". Checkinstall nos hará una serie de preguntas a partir de las cuales instalará el software y creará un paquete que podremos usar para instalarlo en el resto de máquinas de nuestra red.

Veamos paso a paso la compilación y creación de este paquete en la máquina virtual cliente.

Como podéis ver en las líneas que hemos resaltado, nos dice que nos falta el paquete pkg-config o que es demasiado antiguo.

Comprobamos si lo tenemos instalado. 

En este caso, no se encuentra instalado, así que lo instalamos:

   $ sudo apt -y install pkg-config

Una vez instalado, volvemos a ejecutar el script de configuración:

   $ ./configure

Volverá a realizar el chequeo. En este caso, nos aparece un nuevo error (Os mostramos sólo el error): 

   checking for GLIB... no
   configure: error: Package requirements (gio-2.0 >= 2.32.0) were not met:
   No package 'gio-2.0' found

Si buscáis en internet información acerca de gio-2.0 en Ubuntu, veréis que tenemos que instalar el paquete libglib2.0-dev.

Así que lo instalamos:

   $ sudo apt -y install libglib2.0-dev

Una vez instalado, volvemos a ejecutar el script de configuración:

   $ ./configure

Volverá a realizar el chequeo. 

Y comprobamos que siguen faltando cosas por instalar:

   checking for OPENSSL... no
   configure: error: Package requirements (openssl) were not met:
   No package 'openssl' found

Siempre que nos diga que nos falta algún paquete, lo más probable sea que falte alguna librería de desarrollo.

En este caso el paquete openssl se encuentra instalado. Así que, como lo que falta será la librería de desarrollo, busco cuál es el paquete que instala las librerías de desarrollo de openssl y descubro que es libssl-dev. Así que lo instalamos:

   $ sudo apt -y install libssl-dev

Y volvemos a ejecutar el script de configuración:

   $ ./configure

Al ejecutarlo, observamos que nos vuelve a faltar algo más por instalar:

   checking for LIBCURL... no
   configure: error: Package requirements (libcurl) were not met:
   No package 'libcurl' found

Del mismo modo que hicimos anteriormente, buscamos qué librería de desarrollo hay que instalar.

En este caso, en el Ubuntu Jammy donde estamos creando el paquete, comprobamos que para disponer de la librería libcurl, tenemos que instalar el paquete libcurl4-openssl-dev. Así que lo instalamos:

   $ sudo apt -y install libcurl4-openssl-dev

Una vez instalado, volvemos a ejecutar el script de configuración:

   $ ./configure

Al ejecutarlo, comprobamos que sigue faltando algo más:

   configure: error: a2x is required to build docs, install asciidoc

En este caso, nos dice que tenemos que instalar la herramienta asciidoc. Así que la instalamos:

   $ sudo apt -y install asciidoc
