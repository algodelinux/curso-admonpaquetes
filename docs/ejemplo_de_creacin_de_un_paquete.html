Ejemplo de creación de un paquete

Si queréis probar a realizar este ejemplo, necesitaréis dos archivos:

  • linex-icon.png
  • default.xml

Podéis descargar todos los archivos del curso con el siguiente comando:

   $ git clone https://github.com/algodelinux/curso-admonpaquetes.git

Si ya habéis descargado en algún momento los archivos, podéis actualizar los archivos descargados localmente de la siguiente manera:

   $ cd ~/curso-admonpaquetes && git pull

Lo primero que debemos hacer es crear un directorio de trabajo para nuestro paquete:

   $ mkdir -p linex-taskbar-config-algodelinux/debian

Con el parámetro -p estamos creando un directorio con el nombre del paquete y, a la vez, el subdirectorio "debian" donde se alojan los ficheros que nos permitirán gestionar el paquete, entre ellos, el fichero de control.

A continuación, entramos dentro del directorio del paquete:

   $ cd linex-taskbar-config-algodelinux

Y creamos un archivo de control mediante el comando equivs-control:

   $ equivs-control debian/control

Este archivo contendrá un modelo de fichero de control que tendremos que modificar para crear nuestro paquete.

A continuación, editamos el fichero de configuración creado:

   $ nano debian/control

Y tendremos algo como ésto:

   ### Commented entries have reasonable defaults.
   ### Uncomment to edit them.
   # Source: <source package name; defaults to package name>
   Section: misc
   Priority: optional
   # Homepage: <enter URL here; no default>
   Standards-Version: 3.9.2

   Package: <package name; defaults to equivs-dummy>
   # Version: <enter version here; defaults to 1.0>
   # Maintainer: Your Name <yourname@example.com>
   # Pre-Depends: <comma-separated list of packages>
   # Depends: <comma-separated list of packages>
   # Recommends: <comma-separated list of packages>
   # Suggests: <comma-separated list of packages>
   # Provides: <comma-separated list of packages>
   # Replaces: <comma-separated list of packages>
   # Architecture: all
   # Multi-Arch: <one of: foreign|same|allowed>
   # Copyright: <copyright file; defaults to GPL2>
   # Changelog: <changelog file; defaults to a generic changelog>
   # Readme: <README.Debian file; defaults to a generic one>
   # Extra-Files: <comma-separated list of additional files for the doc directory>
   # Links: <pair of space-separated paths; First is path symlink points at, second is filename of link>
   # Files: <pair of space-separated paths; First is file to include, second is destination>
   # <more pairs, if there's more than one file to include. Notice the starting space>
   Description: <short description; defaults to some wise words>
   long description and info
   .
   second paragraph

Lo editamos y dejamos tal que así:

Package: linex-taskbar-config-algodelinux

Version: 1.0
Distribution: jammy
Suite: bionic
Architecture: all
Maintainer: Esteban M. Navas Martín <algodelinux@gmail.com>
Depends: xfce4, xfce4-datetime-plugin, xfce4-whiskermenu-plugin
Replaces: linex-taskbar-config, xubuntu-default-settings
Section: x11
Priority: optional
Changelog: debian/changelog
Links: /usr/share/linex-taskbar-config/default.xml /etc/xdg/xdg-xubuntu/xfce4/panel/default.xml
/usr/share/linex-taskbar-config/default.xml /etc/xdg/xfce4/xfconf/xfce-perchannel-xml/xfce4-panel.xml
Files: usr/share/icons/hicolor/48x48/categories/linex-icon.png /
usr/share/linex-taskbar-config/default.xml /
Description: Configuración de la barra de inicio para linex-Xubuntu
Configuración, applets y apariciencia de la barra de
herramientas en el entorno de trabajo de los clientes
linex para xubuntu.

Para colocar el fichero linex-icon.png, teniendo en cuenta cómo lo hemos especificado en el fichero de control, creamos la ruta donde debe ir el fichero dentro del directorio de trabajo y copiamos el fichero allí:

   $ mkdir -p ~/linex-taskbar-config-algodelinux/usr/share/icons/hicolor/48x48/categories/
   $ cp linex-icon.png ~/linex-taskbar-config-algodelinux/usr/share/icons/hicolor/48x48/categories/

Para colocar el fichero default.xml, teniendo en cuenta cómo lo hemos especificado en el fichero de control, creamos la ruta donde debe ir el fichero dentro del directorio de trabajo y copiamos el fichero allí:

   $ mkdir -p ~/linex-taskbar-config-algodelinux/usr/share/linex-taskbar-config/
   $ cp default.xml ~/linex-taskbar-config-algodelinux/usr/share/linex-taskbar-config/

Vamos a explicar algunos detalles que es importante destacar:

En la línea donde pone Section: hemos dejado x11 porque es un paquete relacionado con el entorno gráfico.

En la línea donde pone Package: escribimos el nombre del paquete:  linex-taskbar-config-algodelinux.

En la línea donde pone Version: especificamos la versión del paquete. En este ejemplo: 1.0.

En la línea Architecture: especificamos la arquitectura para la que va el paquete.

En la línea Depends: especificamos la lista de paquetes que queremos que se instalen, separados por comas: xfce4, xfce4-datetime-plugin, xfce4-whiskermenu-plugin

En la línea Replaces: podemos indicar que un paquete "reemplaza" a otros. Ésto significa que los archivos instalados por el paquete sustituyen a los archivos de los paquetes que se enumeran: linex-taskbar-config, xubuntu-default-settings

En la línea "Changelog: debian/changelog" indicamos a equivs dónde debe localizar el fichero changelog. En el ejemplo, en el subdirectorio debian/changelog.

Para crear el fichero changelog en el directorio debian, estando dentro del directorio del paquete, utilizaremos la herramienta dch:

   $ dch --create --distribution jammy --package "linex-taskbar-config-algodelinux" --newversion 1.0 "First version"

Recordad que si ya habéis creado una versión y lo que estáis haciendo es una nueva versión del paquete, lo que tenéis que hacer será incrementar el número de versión:

   $ dch --increment "Modified to replace files" --distribution bionic

Recordad que cuando incrementéis la versión del paquete en el changelog con dch, también tenéis que reflejar el cambio de versión en el fichero de control.

En la línea "Preinst: debian/preinst" estamos indicando el script que se debe ejecutar antes de instalar el paquete. No olvidéis que si utilizáis un script preinst, tendréis que copiarlo al subdirectorio debian del paquete.

En la línea "Postrm: debian/postrm" indicamos el script que se debe ejecutar después de desinstalar el paquete. También tenemos que copiar el script postrm al subdirectorio debian del paquete.

En este paquete, no necesitamos usar los scripts postinst y prerm, así que no tenemos que usar las directivas correspondientes.

Como ya hemos visto, la línea que dice "Links:" sirve para crear enlaces. El formato para especificar la creación de un enlace es el mismo que cuando creamos un enlace simbólico con ln: Primero se especifica el destino y, a continuación, el enlace. Añadiremos una línea para cada enlace que creemos. Además, es importante destacar que cada nueva línea que cree un link después de la primera, debe ir sangrada, como se puede ver en el ejemplo anterior.

   Links: /usr/share/linex-taskbar-config/default.xml /etc/xdg/xdg-xubuntu/xfce4/panel/default.xml
    /usr/share/linex-taskbar-config/default.xml /etc/xdg/xfce4/xfconf/xfce-perchannel-xml/xfce4-panel.xml

En cuanto a la línea que dice "Files:", nos permite colocar ficheros en la máquina donde se instala el paquetes. El formato es el siguiente: Primero se indica el fichero a colocar, y, a continuación el directorio donde se debe ubicar al instalar el paquete. Del mismo modo que sucede con los links, es importante destacar que cada nueva línea que especifique un fichero, después de la primera, debe ir sangrada, como se puede ver en el ejemplo anterior.

   Files: usr/share/icons/hicolor/48x48/categories/linex-icon.png /
    usr/share/linex-taskbar-config/default.xml /

Una vez creados el fichero de control, los scripts y colocados los ficheros en las ubicaciones desde donde se deben tomar para crear el paquete, tan sólo tendremos que ejecutar, desde el directorio del paquete, el comando que crea el código fuente:

   $ equivs-build debian/control

   dh_autoreconf
   create-stamp debian/debhelper-build-stamp

   dh_install
   dh_installdocs

   dpkg-genbuildinfo --build=binary -O../linex-taskbar-config-algodelinux_1.0_amd64.buildinfo
   dpkg-genchanges --build=binary -O../linex-taskbar-config-algodelinux_1.0_amd64.changes
