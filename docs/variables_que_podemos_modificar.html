Variables que podemos modificar

Cada variable definida en el fichero de configuración tiene un valor por defecto. 

Veamos a continuación para qué sirve cada una de las variables definidas en este fichero de configuración:

Os comentamos lo que nos parece interesante destacar y si tenéis alguna duda, no dejéis de poneros en contacto con vuestro tutor para preguntarle.

  • IGNORE_MUSTHAVE: Esta variable hace que pkgsync ignore el fichero /etc/pkgsync/musthave. De este modo, cuando se ejecute pkgsync se utilizará el fichero /etc/pkgsync/musthave.ies junto con los ficheros definidos en el directorio /etc/pkgsync/musthave.d. Fue una de las primeras opciones de configuración que incluimos en el fichero /etc/default/pkgsync y se hizo para evitar problemas de gestión compartida. Su valor por defecto es "no". Si quisiéramos que pkgsync ignorara el fichero /etc/pkgsync/musthave, tan sólo tendríamos que cambiar el valor a "yes".

  • IGNORE_MAYHAVE: Esta variable hace que pkgsync ignore el fichero /etc/pkgsync/mayhave. De este modo, al ejecutarse pkgsync se utilizará el fichero /etc/pkgsync/mayhave.ies junto con los ficheros definidos en el directorio /etc/pkgsync/mayhave.d. Su valor por defecto es "no". Si quisiéramos que pkgsync ignorara el fichero /etc/pkgsync/musthave, tan sólo tendríamos que cambiar el valor a "yes".

  • IGNORE_MAYNOTHAVE: Esta variable hace que pkgsync ignore el fichero /etc/pkgsync/maynothave. Así, cuando se ejecute pkgsync se utilizará el fichero /etc/pkgsync/maynothave.ies junto con los ficheros definidos en el directorio /etc/pkgsync/maynothave.d.

  • ENABLE: La variable ENABLE permite habilitar o deshabilitar pkgsync. Su valor por defecto es "yes" y podemos cambiarlo a "no" para que no se ejecute. Pkgsync se puede ejecutar automáticamente o manualmente. Si se encuentra deshabilitado, no se ejecutará ni manual ni automáticamente. En este caso, podremos forzar la ejecución utilizando el parámetro -f: "pkgsync -f". Ésto puede ser útil, por ejemplo, en portátiles, que no se encuentran permanentemente conectados a la red. 

  • PURGE_OLD_KERNELS: Ubuntu no elimina los kernels antiguos que se hayan ido instalando en el sistema. Esta opción nos permite eliminar kernels que ya no vamos a usar y liberar espacio. El valor por defecto de esta variable es "no". Si lo activamos cambiando su valor a "yes", por defecto, eliminará todos los kernels instalados salvo los dos últimos.

  • KEEP_LAST_KERNELS: Esta variable nos permite indicar el número de kernels que deben mantenerse en el sistema al realizar una eliminación de kernels antiguos. Su valor por defecto es "2". Podemos incrementarlo, pero no os recomendamos reducirlo. Tener al menos dos kernels, os permitirá arrancar la máquina con un kernel en el caso de que el otro falle.

  • CLEAN: Su valor por defecto es "no". Si lo cambiamos a "yes", cuando se ejecute pkgsync, se eliminarán dependencias de paquetes desinstalados, se purgarán los paquetes desinstalados y se limpiará la caché de paquetes.

  • LAUNCH_SINC_PUPPET: El valor por defecto de esta variable es "yes" para que se realice una sincronización de puppet antes de instalar paquetes. Como nosotros distribuimos las listas de paquetes de pkgsync mediante puppet, nos interesa realizar una sincronización de puppet antes de instalar paquetes. Ahora bien, es posible que alguien quiera usar pkgsync, pero no utilice puppet. En ese caso, tan sólo tendría que cambiar el valor de LAUNCH_SINC_PUPPET a "no". 

  • La variable PUPPET_SERVER permite poder establecer el servidor puppet con el que se va a sincronizar el cliente.

  • Para evitar que pkgsync se quede bloqueado indefinidamente si sinc_puppet no termina de ejecutarse, podemos establecer un TIMEOUT en minutos en la variable TIMEOUT_FOR_SINC_PUPPET.

  • AUTOMATIC_SHUTDOWN_BETWEEN: Al especificar un intervalo horario en esta variable, el equipo se apagará automáticamente al finalizar un pkgsync si la hora en la que se ejecuta se encuentra dentro del intervalo.

  • AUTOMATIC_REBOOT_BETWEEN: Permite especificar un intervalo horario en el que el equipo se reiniciará automáticamente al finalizar pkgsync.

  • AUTOMATIC_REBOOT_INTO_WINDOWS: El valor por defecto es "no". Si lo cambiamos a "yes" y se encuentra definido un intervalo de reinicio en AUTOMATIC_REBOOT_BETWEEN, al realizar el reinicio, el equipo se iniciará en Windows. Esta opción funciona en equipos con UEFI.

  • LAUNCHPAD_GETKEYS: El valor por defecto de esta variable es "yes". Podemos desactivarlo cambiando el valor a "no". Si se encuentra activado, launchpad-getkeys tratará de obtener automáticamente las claves públicas de los repositorios que falten o se encuentren desactualizadas.

  • LAUNCHPAD_GETKEYS_PROXY: En esta variable podemos definir el proxy que launchpad-getkeys usará para obtener las claves. Por defecto, tiene definido el valor "http://servidor:3128", aunque podéis desactivarlo, especificando LAUNCHPAD_GETKEYS_PROXY="". El valor por defecto que hemos definido permite que launchpad-getkeys funcione en nuestros centros, puesto que todos tenemos un proxy squid.

  • En las últimas versiones de launchpad-getkeys, realizamos unas modificaciones que permiten eliminar las claves de repositorios que hayan expirado. Si establecemos  LAUNCHPAD_GETKEYS_REMOVE_EXPIRED_KEYS a "yes", pkgsync eliminará claves de repositorios expiradas.

  • TIMEOUT_FOR_DPKG_OR_APT: Si se está ejecutando dpkg, apt, apt-get o aptitude, pkgsync esperará el tiempo definido en esta variable antes de realizar una actualización de paquetes. Si transcurrido este tiempo, alguna de las herramientas anteriores sigue en ejecución, pkgsync terminará. Esta variable sirve para evitar que pkgsync quede bloqueado por un fallo anterior en la actualización de paquetes. Puede definirse un tiempo en segundos (por ejemplo, 30 o 30s), en minutos (por ejemplo, 10m), en horas (por ejemplo, 6h) o días (1 día). El valor por defecto son 3 minutos.

  • Pkgsync usa aptitude para instalar/desinstalar paquetes. A partir de una determinada versión, los desarrolladores de aptitude realizaron una modificación que hace que aptitude aborte cuando se le pide realizar una instalación de un paquete que no se encuentra en los repositorios. Ésto es un efecto indeseable para nosotros. Para evitar ese problema, podemos usar la variable ABORT_ON_FAILED_REPOS. Simplemente estableciendo ABORT_ON_FAILED_REPOS a "no", lograremos que pkgsync evite este problema.

  • La opción AUTO_TEST_FILES permite habilitar/deshabilitar la revisión automática de los ficheros de listas de paquetes.

  • La opción ALERT_EMAIL_RECEIVER permite definir la dirección de correo electrónico a la que enviar notificaciones.

  • La opción ALERT_EMAIL_ON_ERROR permitirá habilitar/deshabilitar que se envíe un email cuando se produzca un error al realizar un pkgsync. Para que se envíen las alertas, es necesario tener configurado un servicio como postfix o ssmtp en el equipo.

  • La opción ENSURE_ESSENTIAL permite controlar la desinstalación de paquetes esenciales. Por no ser invasivos, la opción por defecto es "no". Nuestra recomendación es establecer el valor a "yes" para garantizar que los paquetes esenciales no se desinstalen.
