Añadir ppa a unattended-upgrades

En la sección anterior, hemos visto cómo añadir patrones al bloque Unattended-Upgrade::Allowed-Origins de unattended-upgrades para  que este servicio realice instalaciones desatendidas desde otros repositorios:

Unattended-Upgrade::Allowed-Origins {
       "${distro_id}:${distro_codename}";
       "${distro_id}:${distro_codename}-security";
// Extended Security Maintenance; doesn't necessarily exist for
// every release and this system may not have it installed, but if
// available, the policy for updates is such that unattended-upgrades
// should also install from here by default.
       "${distro_id}ESMApps:${distro_codename}-apps-security";
       "${distro_id}ESM:${distro_codename}-infra-security";
//     "${distro_id}:${distro_codename}-updates";
//     "${distro_id}:${distro_codename}-proposed";
//     "${distro_id}:${distro_codename}-backports";
       "Google LLC:stable";
};

Aunque el procedimiento es el mismo, vamos a ver, a modo de ejemplo cómo añadir el ppa de Gimp al bloque Unattended-Upgrade::Allowed-Origins para que unattended-upgrades pueda realizar instalaciones desatendidas también desde este repositorio.

Si ejecutáis el comando:

   # unattended-upgrades --dry-run --debug

Veréis que, en la salida del comando, se muestra una línea similar a la siguiente en la que se muestran cuáles son las fuentes desde las que unattended-upgrades va a actualizar automáticamente los paquetes:

Fuentes permitidas: o=Ubuntu,a=jammy, o=Ubuntu,a=jammy-security, o=UbuntuESMApps,a=jammy-apps-security, o=UbuntuESM,a=jammy-infra-security, o=Google LLC,a=stable

Entre las fuentes permitidas, se encuentran los repositorios de Google porque los añadimos anteriormente.

Para añadir un nuevo ppa nuevos repositorios a la sección anterior utilizaremos los siguientes campos:

  • Origin.
  • Codename.

Y tendremos que especificar ambos datos separados por ":"

Para empezar, nos tenemos que fijar en el fichero InRelease de Gimp que se encuentra en el fichero /var/lib/apt/lists/:

head /var/lib/apt/lists/*gimp*InRelease
-----BEGIN PGP SIGNED MESSAGE-----
Hash: SHA512

Origin: LP-PPA-ubuntuhandbook1-gimp
Label: GIMP Image Editor
Suite: jammy
Version: 22.04
Codename: jammy
Date: Wed, 01 Mar 2023 6:58:07 UTC
Architectures: amd64 arm64 armhf i386 ppc64el riscv64 s390x

Tomamos nota del valor de Origin y Codename y los añadimos al bloque Unattended-Upgrade::Allowed-Origins del fichero /etc/apt/apt.conf.d/50unattended-upgrades:

Unattended-Upgrade::Allowed-Origins {
          "${distro_id}:${distro_codename}";
          "${distro_id}:${distro_codename}-security";
   // Extended Security Maintenance; doesn't necessarily exist for
   // every release and this system may not have it installed, but if
   // available, the policy for updates is such that unattended-upgrades
   // should also install from here by default.
          "${distro_id}ESMApps:${distro_codename}-apps-security";
          "${distro_id}ESM:${distro_codename}-infra-security";
   //     "${distro_id}:${distro_codename}-updates";
   //     "${distro_id}:${distro_codename}-proposed";
   //     "${distro_id}:${distro_codename}-backports";
          "Google LLC:stable";
          "LP-PPA-ubuntuhandbook1-gimp:jammy";
};

Si volvéis a ejecutar el comando:

   # unattended-upgrades --dry-run --debug

Veréis que ahora aparece el repositorio de Google como una fuente permitida en unattended-upgrades:

Fuentes permitidas: o=Ubuntu,a=jammy, o=Ubuntu,a=jammy-security, o=UbuntuESMApps,a=jammy-apps-security, o=UbuntuESM,a=jammy-infra-security, o=Google LLC,a=stable, o=LP-PPA-ubuntuhandbook1-gimp,a=jammy

Por tanto, a partir de ahora, los paquetes del ppa de Gimp, podrán actualizarse automáticamente mediante unattended-upgrades.
