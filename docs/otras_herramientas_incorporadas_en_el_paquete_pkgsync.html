Otras herramientas incorporadas en el paquete pkgsync

Obtener el listado de dependencias de un paquete

El paquete pkgsync nos proporciona un script get-package-dependencies al que pasamos como parámetro un nombre de paquete y nos proporciona un listado de las dependencias que tiene ese paquete.

Ésto puede ser muy útil para saber qué paquetes no es necesario añadir a los ficheros de listas de paquetes de pkgsync puesto que ya se instalarán como dependencias de un paquete.

Por ejemplo: Si ejecutamos el comando "get-package-dependencies libreoffice", el script nos mostrará qué paquetes se instalan como dependencia:

   $ get-package-dependencies libreoffice

   libreoffice-base
   libreoffice-calc
   libreoffice-core
   libreoffice-draw
   libreoffice-impress
   libreoffice-math
   libreoffice-report-builder-bin
   libreoffice-writer
   python3-uno

Es decir, que, si añadimos el nombre del paquete libreoffice a cualquiera de los ficheros musthave, no será necesario añadir ninguno de los paquetes mostrados por el script, puesto que éstos se instalarán automáticamente como dependencias del paquete libreoffice.

Descargar un paquete y sus dependencias

Otra de las herramientas que acompañan a pkgsync es el script download_pkg_and_deps, al que tan sólo tenemos que pasar como parámetro un nombre de paquete y descargará el paquete junto con sus dependencias.

Ésto puede ser muy útil, por ejemplo, para agregar los paquetes a un repositorio local o instalarlos manualmente.

Por ejemplo: Si ejecutamos el comando "download_pkg_and_deps libreoffice", el script creará un directorio con el nombre del paquete (en este caso, libreoffice) y descargará dentro de él, el paquete junto con sus dependencias:

Eliminar claves de repositorios expiradas

El script apt-remove-expired-keys elimina las claves de repositorios expiradas.

   # apt-remove-expired-keys

Si no hay claves expiradas, no mostrará ningún resultado  en pantalla.

Realizar un backup de los repositorios

El script backup-repositories permite realizar un backup de los repositorios almacenados en /etc/apt/sources.list y los alojados en el directorio /etc/apt/sources.list.d/ y los almacena en el fichero sources.list.tar.gz alojado en el directorio /var/backups/

   # backup-repositories

Restaurar el backup de los repositorios

El script restore-backup-repositories permite restaurar el backup de los repositorios (/var/backups/sources.list.tar.gz) en el directorio /etc/apt

   # restore-backup-repositories

Obtener un listado de los repositorios que fallan

Para saber si alguno de los repositorios alojados en /etc/apt/sources.list.d/ falla, podemos usar el script list-failed-repositories.

   # list-failed-repositories
   /etc/apt/sources.list.d/desarrollo.educarex.es-jammy.list
   /etc/apt/sources.list.d/repositorio-local-jammy.list

En este ejemplo, están fallando dos repositorios internos porque hemos ejecutado el comando fuera de intranet y no son accesibles desde fuera.

Desactivar repositorios que estén fallando

El script disable-failed-repositories nos permite deshabilitar los repositorios que se encuentran definidos dentro del directorio /etc/apt/sources.list.d/ y que están fallando. Para deshabilitarlos, lo que hace es añadir a los ficheros .list la extensión .disabled

   # disable-failed-repositories

Si al ejecutar el comando list-failed-repositories, obtenéis una lista de repositorios que fallan:

   # list-failed-repositories
   /etc/apt/sources.list.d/desarrollo.educarex.es-jammy.list
   /etc/apt/sources.list.d/repositorio-local-jammy.list

Y los  deshabilitáis:

   # disable-failed-repositories

Podéis comprobar que están deshabilitados simplemente buscando ficheros con extensión .disabled dentro del directorio /etc/apt/sources.list.d/

   # find /etc/apt/sources.list.d/ -name "*.disabled"
   /etc/apt/sources.list.d/desarrollo.educarex.es-jammy.list.disabled
   /etc/apt/sources.list.d/repositorio-local-jammy.list.disabled

Habilitar repositorios deshabilitados

El script enable-disabled-repositories se encarga de habilitar repositorios deshabilitados en /etc/apt/sources.list.d/ quitando a los ficheros .list la extensión .disabled

   # enable-disabled-repositories

Habilitar repositorios deshabilitados que no fallen

Como ya hemos dicho, el script enable-disabled-repositories habilita todos los repositorios que tengan extensión .disabled en el directorio /etc/apt/sources.list.d/

Por el contrario, el script enable-non-failed-repositories habilitará sólo aquellos repositorios deshabilitados en /etc/apt/sources.list.d/ que no fallen quitando a los ficheros .list la extensión .disabled

   # enable-non-failed-repositories

Listar repositorios desactivados

El script list-diabled-repositories muestra un listado de los ficheros de los repositorios alojados en el directorio /etc/apt/ deshabilitados con extensión .disabled

   # list-disabled-repositories

Eliminar todos los repositorios definidos en /etc/apt/sources.list.d

El script remove-all-repositories permite eliminar todos los ficheros de repositorios definidos en el directorio /etc/apt/sources.list.d, se encuentren o no desactivados.

   # remove-all-repositories

Eliminar los repositorios definidos en /etc/apt/sources.list.d que estén fallando

El script remove-failed-repositories elimina de /etc/apt/sources.list.d/ los ficheros de repositorios que están fallando.

   # remove-failed-repositories

Eliminar el backup de repositorios

El script remove-backup-repositories elimina el fichero de backup de los repositorios sources.list.tar.gz alojado en el directorio /var/backups/

   # remove-backup-repositories

Obtener un listado de los paquetes esenciales instalados en el sistema

El script list-essential-packages nos permite obtener un listado de los paquetes esenciales instalados en el sistema. Los paquetes esenciales son los marcados como required, important y standard.

  • Required: Son paquetes necesarios para el propio funcionamiento del sistema.
  • Important: Son paquetes que deben encontrarse en cualquier sistema tipo Unix.
  • Standard: Son paquetes que son estándar en cualquier sistema Linux.
   # list-essential-packages

Actualizar el software instalado en el sistema

Al realizar un pkgsync, antes de instalar/desinstalar el software definido en los ficheros de listas de paquetes, una de las primeras cosas que se hacen es actualizar el software instalado en el sistema.

Ahora bien, para que también podamos actualizar el software del sistema sin realizar un pkgsync, agregamos un script muy interesante a pkgsync, cuyo nombre es softwareupdate, que nos permite actualizar los paquetes instalados mediante apt, flatpak, snap, además de realizar limpieza de paquetes al terminar.

No os preocupéis si en vuestro sistema no tenéis instalados paquetes flatpak o snap porque el script tan sólo tratará de actualizar este tipo de paquetes si la herramienta flatpak o la herramienta snap se encuentran instaladas.

Si queréis actualizar la paquetería de vuestro sistema, en lugar de hacer un simple "apt upgrade", os recomendamos usar este script porque, antes de actualizar, tratará de obtener las claves de repositorios que falten pero además, intentará resolver problemas de paquetes.

   # softwareupdate

Si queréis que softwareupdate apague el equipo al terminar, podéis pasarle el parámetro -S:

   # softwareupdate -S

Y si queréis que se reinicie el equipo al terminarl, podéis pasarle el parámetro -R:

   # softwareupdate -R

Si no recordáis la sintaxis de softwareupdate, siempre podéis ejecutarlo con el parámetro -h:

   # softwareupdate -h
   softwareupdate 1.2
   Actualizar el software del equipo

   Uso: /usr/local/sbin/softwareupdate [OPTIONS]
   Opciones reconocidas:
   -h mostrar esta ayuda y salir.
   -S apagar el equipo al finalizar el proceso de actualización.
   -R reiniciar el equipo al finalizar el proceso de actualización.

Resolver problemas de paquetería

Si existen problemas en el sistema de paquetes, tanto pkgsync como softwareupdate van a tratar de resolverlos. No obstante, hemos agregado también un script solve-apt-errors que trata de resolver posibles errores de apt para poder ejecutarlo también independientemente de pkgsync o softwareupdate.

   # solve-apt-errors

Si no existen errores en el sistema de paquetes, el script no hará nada.

Esperar mientras se esté trabajando con el gestor de paquetes

Por último, comentar que el paquete pkgsync nos proporciona un script auxiliar wait_for_apt_or_dpkg que nos permite realizar una espera activa mientras alguna herramienta está usando el gestor de paquetes. Este script lo utilizamos, por ejemplo, en pkgsync, softwareupdate o solve-apt-errors.
